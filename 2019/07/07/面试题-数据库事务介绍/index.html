面试题|mysql数据库事务介绍

数据库
  • mysql
  • 事务
  • 面试题

为什么要使用事务

因为我们针对数据库的操作有些时候需要多个语句同时执行成功，并且不能被其他线程操作干扰，保证数据和预期一样。

事务的性质

原子性（A） 构成事务的的所有操作必须是一个逻辑单元，要么全部执行，要么全部不执行。
稳定性（C） 数据库在事务执行前后状态都必须是稳定的。
隔离性（I） 事务之间不会相互影响。
持久性（D） 事务执行成功后必须全部写入磁盘。

事务的级别

Read UnCommitted(未提交读)

一个事务读取到了另一个事务还未提交的数据。也就是读取到了脏数据。该级别会出现脏读，不可重复读，幻读

Read Committed(已提交读)

一个事务读取到另一个事务已经提交的数据。该级别会出现不可重复读，幻读

Repeatable Read(可重复读)

一个事务开启后可以对相同的数据进行重复读取。该级别会出现幻读

Serializable(可串行化)

该级别相当于串行执行事务。

事务中的读取数据问题

脏读

脏读就是指当一个事务正在访问数据，并且对数据进行了修改，而这种修改还没有提交到数据库中，这时，另外一个事务也访问这个数据，然后使用了这个数据。

不可重复读

是指在一个事务内，多次读同一数据。在这个事务还没有结束时，另外一个事务也访问该同一数据。那么，在第一个事务中的两次读数据之间，由于第二个事务的修改，那么第一个事务两次读到的的数据可能是不一样的。这样就发生了在一个事务内两次读到的数据是不一样的，因此称为是不可重复读。

幻读

第一个事务对一个表中的数据进行了修改，这种修改涉及到表中的全部数据行。同时，第二个事务也修改这个表中的数据，这种修改是向表中插入一行新数据。那么，以后就会发生操作第一个事务的用户发现表中还有没有修改的数据行，就好象发生了幻觉一样。

总结

我们用表格来整理下

隔离级别 脏读 不可重复读 幻读 锁
未提交读 Y Y Y 不加锁
读已提交 N Y Y 加读锁
可重复读 N N Y 加读写锁
可串行化 N N N 表级共享锁，读写阻塞

事实上mysql并不是简单的对事务加锁，为了更高的读写并发，通常使用mcvv多版本控制增加读写并发。这将涉及到redo log undo log使用。
